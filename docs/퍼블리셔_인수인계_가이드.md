# 서울이도치과 홈페이지 — 퍼블리셔 인수인계 가이드

> 대상: 본 사이트를 넘겨받아 **PHP(MPA)** 로 개발할 퍼블리셔/개발자
> 작성 기준일: 2026-06-26
> 현재 상태: 개원 전 디자인 초안(`noindex`), GitHub Pages에 정적 배포되어 동작 중

---

## 0. 30초 요약

- **이건 SPA가 아닙니다.** React/Vue/빌드 번들러 **없음**. 순수 **HTML + CSS + 바닐라 JS**로 만든 **정적 멀티페이지(MPA)** 입니다.
- 페이지는 실제 파일 4개로 존재합니다: `index.html`(ko) / `en/index.html` / `ja/index.html` / `zh/index.html`.
- 한국어 `index.html`이 **유일한 원본**이고, 나머지 3개 언어는 `scripts/gen-lang-pages.js` Node 스크립트가 i18n 딕셔너리로 **빌드 타임에 자동 생성**한 산출물입니다.
- JS가 하는 일은 "보조"뿐입니다: 인트로 영상, 스크롤 애니메이션, 모바일 메뉴 토글, 다국어 텍스트 치환. **콘텐츠를 JS로 그리지 않습니다** — HTML 안에 이미 다 있습니다(서버 렌더 친화적).
- **PHP 전환은 구조 변경이 아니라 "정리"입니다.** 반복 영역(head/nav/footer)을 `include`로 빼고, i18n JS 딕셔너리를 PHP 배열로 옮기면 끝. Node 빌드 스크립트는 PHP 서버 렌더링으로 대체되며 사라집니다.

## 1. 기술 스택 (있는 그대로)

| 항목 | 내용 |
|---|---|
| 프레임워크 | **없음** (no React/Vue/Svelte) |

## 2. 폴더 구조

> ⚠️ **루트는 곧 라이브 배포 루트입니다.** 이 폴더가 GitHub Pages(`half1126-byte.github.io`) 저장소이고, 사용자 루트 Pages는 **브랜치 루트에서 그대로 서빙**합니다. 따라서 `index.html`·`css/`·`js/`·`assets/`·`video/`·`i18n/`·`en/ja/zh`는 **반드시 루트에 있어야** 라이브가 안 깨집니다(하위 폴더로 옮기지 말 것).

├── index.html              ← ★ 한국어 원본(유일한 SoT). 모든 콘텐츠가 여기 있음

`★` = 손댈 일이 많은 핵심 파일. **①은 루트 고정**, ②는 PHP 호스팅용, ③은 비배포(자료/·제작물/은 비공개).

> **Jekyll 주의**: GitHub Pages는 기본 Jekyll이라 `_`로 시작하는 폴더(`_작업본/`)를 무시합니다. 루트의 빈 `.nojekyll` 파일이 Jekyll을 꺼서 모든 폴더를 그대로 서빙하게 합니다(정적 사이트 표준).
>
> **공개/비공개**: `자료/`·`제작물/`는 `.gitignore`로 **공개 저장소에 올라가지 않습니다**(브랜드 원본·미팅노트 보호). 반면 `docs/`는 추적되어 공개되니 기밀 문서를 넣지 마세요.

## 4. 다국어(i18n) 시스템 — **여기가 핵심**

PHP 전환에서 가장 신경 쓸 부분이라 원리를 먼저 설명합니다.

### 4-1. 현재 동작 방식

HTML 요소에 `data-i18n` 계열 속성으로 "번역 키"를 박아둡니다:

```html
<!-- 텍스트 치환 -->
<h2 data-i18n="services.title">진료과목</h2>

<!-- 인라인 HTML 포함 치환(<br>, <span> 등 신뢰된 마크업) -->
<h1 data-i18n-html="hero.title">바른 길을 걷다,<br><span class="accent">완벽에 닿다.</span></h1>

<!-- 속성 치환(aria-label, alt 등) — "속성명:키" 세미콜론 구분 -->
<button data-i18n-attr="aria-label:nav.menu;title:nav.menu">…</button>
```

키-값 사전은 `i18n/ko.js` 등에 평면 객체로 있습니다:

```js
window.YIDO_I18N.ko = {
  "services.title": "진료과목",
  "hero.title": "바른 길을 걷다,<br><span class=\"accent\">완벽에 닿다.</span>",
  ...
};
```

**폴백 체인:** 선택 언어 값이 비어있으면 → 한국어 → 그래도 없으면 HTML 원문 유지. (번역 누락이 화면 깨짐으로 이어지지 않음)

### 4-2. 두 가지 렌더 경로 (이게 SPA처럼 보이는 이유)

- **배포(http/https):** 언어 버튼 클릭 → 실제 `/en/` 같은 **정적 페이지로 이동**. 각 언어 페이지는 이미 해당 언어로 프리렌더되어 있음(JS 없이도 완성된 HTML). → **정상 MPA, SEO 친화적**.
- **로컬 `file://` 미리보기:** 페이지 이동 없이 `js/i18n.js`가 DOM 텍스트만 즉석 교체. (개발 편의용)

`scripts/gen-lang-pages.js`가 ko `index.html`을 읽어 `data-i18n` 키를 각 언어 값으로 치환하고, `<title>`·메타·canonical·hreflang·자산 절대경로까지 바꿔 `/en /ja /zh/index.html`을 만들어냅니다.

### 4-3. ⚠️ 콘텐츠 수정 시 현재 규칙

> 한국어 문구를 고칠 때는 **`index.html`과 `i18n/ko.js` 두 곳을 함께** 고치고, 반드시 `node scripts/gen-lang-pages.js`를 재실행해 다른 언어 페이지를 다시 생성해야 합니다. (이 이중관리가 정적 사이트의 유일한 번거로움 — PHP로 가면 해소됩니다)

## 6. PHP(MPA) 전환 — **이미 동작하는 변환본이 들어 있습니다** ✅

이 폴더에는 **작동하는 PHP MPA 골격이 이미 포함**되어 있습니다. **partials(head/header/footer) + pages(본문) 조립** 방식으로, 라우터가 `/{lang}/{pageKey}/`를 받아 페이지를 서버 렌더합니다. (검증: 홈 ko/en/ja 17섹션·JSON-LD 3블록 전부 유효, 자산 전부 절대경로, 404 동작)

> 멀티페이지 확장 설계는 [멀티페이지_확장계획.md](멀티페이지_확장계획.md) 참고. 페이지 추가 = `config/pages.php`에 1줄 + `pages/<page>.php` + 번역 키 + (선택) 스키마 파일.

### 6-0. PHP 파일 구성 (그대로 쓰면 됨)

| 파일 | 역할 |
|---|---|
| `index.php` | 라우터: `/{lang?}/{pageKey}/` 파싱 → `config/pages.php` 매핑 → head+header+page+footer 조립 |
| `config/pages.php` | ★ 라우트 화이트리스트(페이지키 → 템플릿·메타키·스키마). **새 페이지는 여기에 추가** |
| `partials/head.php` | 페이지·언어별 `<head>` 생성(메타·canonical·hreflang·CSS·JSON-LD). 자산은 절대경로 |
| `partials/header.php · footer.php` | 전역 헤더/네비·푸터·스크립트(프래그먼트, `data-i18n` 서버 해석) |
| `partials/_schema_home.html` | 홈 JSON-LD(Dentist·FAQPage·WebPage). 페이지별 스키마는 파일 추가 |
| `pages/home.php` | 홈 본문(`<main>`). 새 페이지는 `pages/<page>.php`(또는 `pages/treatments/*.php`) |
| `lib/i18n.php` | i18n 엔진(`resolve`·`apply_i18n`·`rewrite_assets`) + `t()` 헬퍼 |
| `lang/ko·en·ja·zh.php` | 번역 배열(각 278키). `i18n/*.js`에서 자동 변환 |
| `scripts/gen-php-lang.js` | `i18n/*.js` → `lang/*.php` 재생성기 |
| `.htaccess` / `serve.php` | Apache 운영(모든 경로→라우터) / 로컬 QA(PHP 내장 서버) |

### 6-0-1. 실행 방법

```bash
# 로컬 미리보기 (PHP 7.4+ / 8.x)
php -S localhost:8000 serve.php
#   http://localhost:8000/      → 한국어
#   http://localhost:8000/en/   → 영어 (ja/ zh/ 동일)

# 콘텐츠(문구) 수정 후 번역배열 재생성
node scripts/gen-php-lang.js     # i18n/*.js → lang/*.php
```

운영 서버(Apache+PHP)는 `.htaccess`가 `/en/ /ja/ /zh/`를 `index.php`로 넘깁니다. Nginx면 `try_files $uri $uri/ /index.php?lang=$1;` 식 location 블록으로 동일하게 처리하세요.

### 6-0-2. 동작 원리

`index.php` → URL에서 언어(`/en|ja|zh/`)와 페이지키 분리 → `config/pages.php`에서 페이지 설정 조회(없으면 404) → `lang/<lang>.php`+`lang/ko.php`(폴백) 로드 → **`partials/header.php` + `pages/<template>` + `partials/footer.php`** 를 모아 `data-i18n`를 서버 해석 → **`partials/head.php`**(메타·canonical·hreflang·스키마, 언어별 `t()`)와 합쳐 출력. **클라이언트 JS 없이 각 언어가 완전한 HTML로 렌더**됩니다(추출성·GEO 유리).

새 페이지 추가 절차:
1. `config/pages.php`에 항목 추가(`path`·`template`·메타키·`schema_file`).
2. `pages/<page>.php`에 본문 작성(`data-i18n="키"` 사용 → 번역 자동).
3. `i18n/*.js`(또는 `i18n/pages/<page>.js`)에 키 추가 → `node scripts/gen-php-lang.js`.
4. (선택) `partials/<page>_schema.html`에 JSON-LD.

## 6.5 (참고) 본문을 `<?= t('키') ?>`로 더 PHP답게

현재 본문 프래그먼트(`pages/*.php`)는 `data-i18n` 마커를 서버가 해석합니다(원본 HTML 그대로라 안전). 팀 컨벤션상 본문에서 `<?= t('키') ?>`를 직접 부르고 싶다면 아래를 참고하세요. `t()`는 이미 제공됩니다.

### 6-1. 디렉터리(권장)

```
/
├── index.php                 ← 라우팅 진입(언어 감지 → 본문 include)
├── includes/
│   ├── head.php              ← <head> 공통(메타·CSS·스키마)
│   ├── header.php            ← 헤더/네비게이션
│   ├── footer.php            ← 푸터 + JS 로드
│   └── i18n.php              ← 번역 배열 + t() 헬퍼
├── lang/
│   ├── ko.php  en.php  ja.php  zh.php   ← i18n/*.js → PHP 배열로 이관
├── pages/
│   └── home.php              ← 본문 섹션(현재 index.html <body> 내용)
├── css/  js/  assets/  video/   ← 그대로 복사(수정 불필요)
```

### 6-2. i18n: JS 딕셔너리 → PHP 배열

`i18n/ko.js`의 객체를 거의 그대로 PHP 배열로 옮기면 됩니다:

```php
// lang/ko.php
return [
  'services.title' => '진료과목',
  'hero.title'     => '바른 길을 걷다,<br><span class="accent">완벽에 닿다.</span>',
  // ...
];
```

```php
// includes/i18n.php
function t($key) {
  global $DICT, $DICT_KO;
  if (!empty($DICT[$key]))    return $DICT[$key];   // 선택 언어
  if (!empty($DICT_KO[$key])) return $DICT_KO[$key]; // 한국어 폴백
  return '';                                         // 없으면 빈값
}
```

그리고 HTML의 `data-i18n="services.title"` 자리를 서버 출력으로 바꿉니다:

```php
<!-- before (정적) -->
<h2 data-i18n="services.title">진료과목</h2>
<!-- after (PHP, 텍스트는 escape) -->
<h2><?= htmlspecialchars(t('services.title'), ENT_QUOTES, 'UTF-8') ?></h2>

<!-- data-i18n-html 자리는 신뢰된 인라인 마크업이라 escape 없이 출력 -->
<h1><?= t('hero.title') ?></h1>
```

> **이관 팁:** `i18n/*.js`는 `window.YIDO_I18N.ko = { ... }` 한 덩어리라, 객체 본문만 떼어 `key => value`로 기계 변환하면 됩니다(`:` → `=>`, 끝 `;` → 없음). 키 구조와 값은 완성되어 있으니 **번역을 새로 할 필요 없음**.

### 6-3. 언어 라우팅

```php
// index.php
$lang = preg_match('#^/(en|ja|zh)(/|$)#', $_SERVER['REQUEST_URI'], $m) ? $m[1] : 'ko';
$DICT_KO = require __DIR__."/lang/ko.php";
$DICT    = $lang === 'ko' ? $DICT_KO : require __DIR__."/lang/$lang.php";
require __DIR__."/includes/head.php";
require __DIR__."/includes/header.php";
require __DIR__."/pages/home.php";
require __DIR__."/includes/footer.php";
```

`.htaccess`(Apache)로 `/en/` → `index.php?lang=en` 리라이트하면 현재 URL 구조(`/en/`)를 그대로 유지할 수 있어 SEO·hreflang을 손대지 않아도 됩니다.

### 6-4. 전환 시 사라지는 것 / 유지되는 것

| 사라짐 | 유지 |
|---|---|
| `scripts/gen-lang-pages.js` (서버 렌더가 대체) | `css/`, `assets/`, `video/` 전부 그대로 |
| `en/ ja/ zh/` 사전생성 HTML(서버가 즉석 렌더) | `js/`의 nav·video·scroll·ba 등 보조 스크립트 그대로 |
| `js/i18n.js`의 텍스트 치환 로직 | `js/i18n.js`의 **언어 스위처 클릭 핸들러**(URL 이동)는 유지하거나 PHP `<a href>`로 대체 |
| 콘텐츠 이중관리(html+딕셔너리) | `data-i18n` 키 네이밍 체계(번역 키로 재활용) |
